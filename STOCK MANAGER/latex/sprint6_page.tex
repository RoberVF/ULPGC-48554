Actualmente, nuestro proyecto esta siguiendo una estructura de pruebas por capas, donde TDD y BDD se complementan en lugar de sustitutirse.

En cuanto al TDD, implementado con {\bfseries{PHPUnit}}, buscammos que cada unidad de codigo en si misma, siendo estas clases y metodos, funcionen segun lo previsto. Esto implica por ejemplo que un metodo para insertar productos en la base de datos, esta creando una nueva fila en ella o que un metodo pensado para decrementar el stock de un producto lo esta decrementando correctamente.

Por otra parte, tenemos el BDD, implementado con {\bfseries{Behat}}, lo cual comprueba el flujo de trabajo completo a traves de la metodologia MVC. Esto implica que no solamente se valide el metodo de insertar el producto sino que valide el comportamiento completo del sistema desde la perspectiva del usuario (se integran los datos en el formulario para crear un nuevo producto, se recibe la solicitud en el controlador y el producto aparece correctamente listado en la lista de productos totales de la vista).\hypertarget{sprint6_page_autotoc_md16}{}\doxysubsection{\texorpdfstring{Sobre Mink y sus drivers}{Sobre Mink y sus drivers}}\label{sprint6_page_autotoc_md16}
{\bfseries{Mink}} es una biblioteca de PHP de codigo abierto que actua como un controlador o emulador de navegador para aplicaciones web. Con esta herramienta se pueden simular las interacciones reales de un usuario al "{}clicar en la web directamente"{}.

{\bfseries{Mink}} traducira los pasos del lenguaje Gherkin de Behat (ej., "{}\+Cuando hago clic en el boton \textquotesingle{}\+Incluir Producto\textquotesingle{}"{}) en acciones reales del navegador. Para ello, debemos utilizar un driver para el navegador llamado {\bfseries{Goutte}}. Este driver es un cliente simple HTTP para hacer peticiones sin Javascript, perfecto para nuestro proyecto ya que no estamos utilizando JS.

He visto que existen drivers mas potentes como {\bfseries{Chrome\+Driver}} o {\bfseries{Selenium2\+Driver}} que si que permiten controlar un navegador real como {\itshape Chrome} para ejecutar las pruebas dependientes de Javascript pero no creo que sea necesario utilizar herramientas mas complejas ya que con lo que ofrece {\bfseries{Goutte}} sera suficiente.

Utilizando {\bfseries{Mink}}, la capa BDD hecha con {\bfseries{Behat}} podra validar el flujo completo de la aplicacion, incluyendo ahi la interfaz y no quedandonos solamente en la logica del backend.\hypertarget{sprint6_page_autotoc_md17}{}\doxysubsubsection{\texorpdfstring{Instalacion de Mink y Goutte}{Instalacion de Mink y Goutte}}\label{sprint6_page_autotoc_md17}

 Esto instalara la ultima version actualizada de ambos. Hay que tener en cuenta que he debido de eliminar {\itshape composer.\+lock} para poder realizar la instalacion correctamente.\hypertarget{sprint6_page_autotoc_md18}{}\doxysubsubsection{\texorpdfstring{Configuracion de Mink}{Configuracion de Mink}}\label{sprint6_page_autotoc_md18}
Para configurar {\bfseries{Mink}} he tenido que crear en la raiz del proyecto el archivo {\ttfamily behat.\+yml}. Este archivo sera el que le especifique a {\bfseries{Mink}} en que lugar debe actuar para poder simular la navegacion. Ademas, se debe de especificar cual sera el contexto del proyecto, en nuestro caso {\ttfamily Feature\+Context}, que utilizaremos {\bfseries{Goute}} y cual sera la url base del proyecto.\hypertarget{sprint6_page_autotoc_md19}{}\doxysubsubsection{\texorpdfstring{Problemas en el uso de Mink}{Problemas en el uso de Mink}}\label{sprint6_page_autotoc_md19}
El avance del SPRINT 6 esta bloqueado debido a un conflicto de dependencias irresoluble en {\bfseries{Composer}}. La version actual de {\bfseries{Behat}} (\texorpdfstring{$^\wedge$}{\string^}3.25) requiere librerias de {\bfseries{Symfony}} modernas (v5+), pero la extension de {\bfseries{Mink}} ({\itshape mink-\/extension}) que estoy intentando instalar solo encuentra versiones antiguas que dependen de {\bfseries{Symfony}} v2/v3. {\bfseries{Composer}} no puede trabajar con estas diferentes versiones, lo que impide instalar las herramientas de testing web ({\bfseries{Mink/\+Goutte}}) necesarias para continuar.
